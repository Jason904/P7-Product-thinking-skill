# Hermes Domain Add-ons

Load this reference when the case touches 3D AI, AI psychology, GitHub/open source, Agent/Workflow/Skill/MCP, AI Coding governance, or Eval/Gate.

## 3D AI / 3D Agent / 空间智能

Core judgment:

```text
这不是一次性生成 3D 资产，
而是把空间意图转化为可编辑、可复用、可交付的 3D 工作流。
```

Inspect:

- Who: 3D designers, education creators, industrial training teams, game developers, AR/VR teams, ecommerce display teams, architecture/spatial designers, non-professional creators, sales/marketing teams, payers vs users.
- Where: modeling, scene construction, material/light, animation/interaction, display/share, AR/VR publishing, teaching/training, industrial simulation, digital twin.
- Losses: high modeling cost, high expertise threshold, long cycle, expensive changes, hard collaboration, poor reuse, complex interaction design, limited AR/VR content supply, non-expert users unable to express spatial intent, generated assets not editable/controllable/verifiable.
- Required analysis: 3D 工作流分析, 资产可交付性分析, 空间交互链路分析, 3D Eval.

Trend path:

```text
手动 3D 工具 → AI 辅助生成 3D 资产 → AI 辅助编辑 3D 场景 → 3D Agent 执行空间任务 → 多 Agent 协作生成复杂 3D 场景 → AR / VR / 空间计算内容自动化生产系统
```

Opportunity sentence:

```text
最大机会不在一次性生成一个 3D 模型，
而在把 3D 意图转化为可编辑、可复用、可交付的空间工作流。
```

## 心理测评 / AI 心理效果评价 / 心理评估

Core judgment:

```text
这不是 AI 陪聊，
而是安全、可验证、可分级、可转介的心理支持与评估系统。
```

Inspect:

- Who: self-help users, emotional-stress users, students/parents/teachers, employees, HR/EAP, counseling institutions, medical institutions, insurance/health management, platform trust/safety teams, professional counselors/clinicians.
- Where: emotion management, stress assessment, mental-health screening, educational psychology, workplace psychology, pre-consultation assessment, consultation assistance, post-consultation effect tracking, crisis detection and referral, enterprise EAP.
- Losses: users lack self-knowledge and low-threshold support, professional service is expensive and scarce, evaluation is subjective, effects are hard to quantify, risk detection is weak, privacy/ethics risk is high, AI may over-accommodate or give unsafe advice, crisis users may not be referred.
- Desired gains: low-threshold support, early risk identification, self-awareness, timely emotional relief, standardized assessment, tracked effect changes, safe triage/referral, privacy-controlled entry point.

Effect evaluation must check:

- Short-term emotional improvement
- Self-awareness improvement
- Whether the user feels understood
- Whether advice is safe and appropriate
- Whether the AI avoids over-accommodation
- Whether crisis signals are identified
- Whether human/professional referral is correct
- Whether privacy is protected
- Whether dependency is avoided
- Whether validated scales, experiments, or longitudinal tracking support claims

Safety boundaries:

- 不做临床诊断。
- 不输出医疗级结论。
- 不把 AI 心理陪伴等同于专业心理治疗。
- 不忽略危机识别、隐私、伦理、责任边界和人工转介。
- Distinguish entertainment/companionship, self-help tool, psychological assessment tool, clinical assistive assessment, and medical-grade service.

Trend path:

```text
泛心理陪伴 → AI 自助心理工具 → AI 心理测评和情绪评估 → AI 辅助心理咨询流程 → 人机协同心理服务系统 → 安全、合规、可追踪的心理健康基础设施
```

## GitHub / 开源趋势 / 开发者生态

Core judgment:

```text
这不是一个 repo 火了，
而是开发者在寻找某种新工作流 / 新基础设施 / 新控制点。
```

Do not rely on total stars alone. Inspect:

- Star/fork growth velocity
- Commit, release, issue, PR, discussion activity
- README clarity, docs, demos, examples, evals, benchmarks
- Whether it solves a real developer workflow pain
- Whether it can become a product, platform, protocol, control plane, or enterprise infrastructure
- Community distribution, launch narrative, and demo quality
- Relevance to the user's projects, interviews, personal brand, or Watchlist

## Agent / Workflow / Skill / MCP

Focus on how AI capability becomes repeatable execution:

- Role split between Agent, Workflow, Skill, MCP, tools, memory, and governance.
- Task success criteria, failure handling, rollback, audit, permission, and result acceptance.
- Whether the product is a demo, a one-off agent, a stable workflow, an ability asset, or enterprise AI infrastructure.
- Value control points: tool protocol, workflow orchestration, context gateway, eval harness, policy layer, audit layer, reusable skill library.

## AI Coding 与工程治理

Analyze as an engineering collaboration system, not just code generation:

- Source of truth: current files, tests, CI, rendered behavior, PR state, logs.
- Gates: Plan Gate, Review Gate, QA Gate, regression validation.
- Risks: summary-based hallucination, green tests with broken user path, branch drift, missing rollback, unclear ownership.
- Opportunity: stable AI coding workflows, review harnesses, quality gates, reusable skills, engineering governance.

## Harness / Eval / Gate / AI 质量体系

Separate model capability evaluation from product effect evaluation.

Inspect:

- Unit of evaluation: final answer, step, trace, retrieval evidence, tool call, UI output, business outcome.
- Golden Set, rubric, AI Judge, human review, code validation, regression suite.
- Gate design: admission, release, rollback, audit, monitoring.
- Product value: turning subjective quality into measurable acceptance criteria.
